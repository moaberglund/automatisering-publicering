Arbetsprocess

För att starta upp projektet skapar jag en mapp, sedan kan jag öppna upp terminalen på datorn. För att navigera mig runt i terminalen kan jag skriva cd som står för change directory, följt av vägen till min mapp för att ställa mig i mappen. När jag väl är i önskad mapp skriver jag npm init - och trycker enter för att initiera. Jag kan sedan antingen skriva code . för att öppna Visual Studio Code, alternativt öppna programmet manuellt. Här har nu en package.json fil skapats, i denna tar jag bort ”main” och ”script” då dessa är onödiga.


Nu är paketet redo att installeras. I terminalen skriver jag npm install parcel --save-dev och trycker enter. Här efter är det viktigt att skapa en .gitignore fil för node_modules/. Även dist/ och .parcel-cache kan vara bra att placera här. I package.json kan jag nu även lägga till:
”source”: ”src/index.html” ,
”scripts” : {
”start”: ”parcel”,
”build”: ”parcel build”
},


Därefter kan webbplatsen byggas upp med lämpliga filer. Jag skapar en mapp som heter src, och i denna läggs sedan index filen samt andra mappar med tillhörande filer.


Härifrån går jag till sourcekontroll och väljer att skapa ett nytt repo via Github för webbplatsen.


När repot är skapat på Github kan jag sedan ladda upp det till Netlify. Jag loggar in på deras hemsida, väljer add new site och import an existing project, väljer Github och sedan det repo som ska laddas upp. De filer som laddas upp till Netlify ska vara från dist mappen (publish directory) och kommando för publisering anges som npm run build.


För att optimera bilderna använder jag en picture tagg som sedan inkluderar olika source taggar och i vanlig ordning avslutas med en img tagg som fallback, innehållande alt-attribut. För vidare kontrollering skapar jag även en sharp.config.json fil när jag kan specificera karaktärer för olika bildformat.


För att kontrollera bakåtkompatibiltet av sidan går jag in i package.json filen och lägger till "browserlist": ">0.5%, last 2 versions, not dead"
